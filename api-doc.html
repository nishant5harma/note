🔐 Super Admin Reference

CRM Backend API

Complete endpoint reference for Auth, Users, Roles, and Permissions — scoped to the superadmin role.

Base URL: /api Auth: Bearer <token> Format: application/json
🔑

Auth

POST /api/auth/login No auth required

Request Body

{
  "email": "superadmin@example.com",
  "password": "SuperSecurePassword123!"
}
emailstring · required · valid email
passwordstring · required

Response 200

{
  "accessToken": "eyJhbGciOiJIUzI1...",
  "user": {
    "id": "uuid",
    "name": "Super Admin",
    "email": "superadmin@example.com",
    "roles": [{ "name": "superadmin" }]
  }
}

Refresh token is set as an HttpOnly cookie.

GET /api/auth/me 🔒 requireAuth

Headers

Authorization: Bearer <accessToken>

No request body needed.

Response 200

{
  "user": {
    "id": "uuid",
    "name": "Super Admin",
    "email": "superadmin@example.com",
    "roles": [{ "id": "uuid", "name": "superadmin" }],
    "permissions": ["user.read", "role.write", ...]
  }
}
POST /api/auth/refresh 🔒 requireAuth

Request Body (optional)

{
  "refreshToken": "eyJhbGciOi..."
}

Or pass via refreshToken cookie.

Response 200

{
  "accessToken": "eyJhbGciOiJIUzI1..."
}
POST /api/auth/logout 🔒 requireAuth

Request Body (optional)

{
  "refreshToken": "eyJhbGciOi..."
}

Or pass via refreshToken cookie.

Response 200

{
  "message": "Logged out successfully"
}

Users

Permission required: user.write / user.read
POST /api/users user.write 🔒 requireAuth

Request Body

{
  "name": "John Doe",
  "email": "john.doe@example.com",
  "password": "SecurePass123!",
  "roleIds": ["<role-uuid>"],
  "teamId": "<team-uuid>"
}
namestring · required · min 2
emailstring · required · valid email
passwordstring · required · min 6
roleIdsUUID[] · required · min 1
teamIdUUID · optional · adds as MEMBER

Response 201

{
  "user": {
    "id": "uuid",
    "name": "John Doe",
    "email": "john.doe@example.com",
    "roles": [{
      "id": "uuid",
      "name": "manager",
      "description": "..."
    }],
    "teamMembers": [{
      "teamId": "uuid",
      "teamName": "Sales Team",
      "role": "MEMBER",
      "joinedAt": "2026-03-13T..."
    }]
  }
}
GET /api/users user.read 🔒 requireAuth

Headers

Authorization: Bearer <accessToken>

No request body. Returns up to 100 users ordered by creation date.

Response 200

{
  "users": [{
    "id": "uuid",
    "name": "John Doe",
    "email": "john.doe@example.com",
    "roles": [{ "id": "uuid", "name": "manager" }],
    "teamMembers": [{
      "teamId": "uuid",
      "teamName": "Sales",
      "role": "MEMBER",
      "joinedAt": "2026-03-13T..."
    }]
  }]
}
GET /api/users/:id user.read 🔒 requireAuth

Path Param

:idUUID of the user

Response 200

{
  "user": {
    "id": "uuid",
    "name": "John Doe",
    "email": "john.doe@example.com",
    "roles": [{ "id": "uuid", "name": "manager" }],
    "teamMembers": [...]
  }
}

All Seeded Permissions

These 45 permissions are created by the super-admin seed and assigned to the superadmin role.

User
user.readView user data
user.createCreate users
user.writeWrite users
user.updateUpdate user data
user.deleteDelete users
Role
role.readView roles
role.createCreate roles
role.writeWrite roles
role.updateUpdate roles
role.deleteDelete roles
Permission
permission.readView permissions
permission.assignAssign permissions to roles
permission.writeWrite permissions
permission.deleteDelete permissions
Team
team.readView teams
team.createCreate teams
team.updateUpdate teams
team.deleteDelete teams
team.writeWrite teams (approve assignments/removals)
team.assignLeadAssign team lead
team.requestCreate team membership requests
team.approveApprove team requests
System / AuditLog
auditlog.readView audit logs
system.configAccess system configuration
admin.viewAccess to the Admin Dashboard
HR — Attendance · Location · Device · Consent
attendance.selfPerform own attendance check-in
attendance.readRead attendance records
location.requestCreate on-demand location requests
location.readRead location request results
location.auditView location access audit records
device.registerRegister/update a device (push token)
device.unregisterUnregister device / remove push token
device.readList devices for a user
consent.writeCreate or revoke consent entries
consent.readRead consents for a user
Lead
lead.readRead leads
lead.writeCreate/update leads
lead.assignAssign leads
lead.noteAdd notes to leads
lead.followupManage lead follow-ups
lead.webhookView raw webhook events
Inventory
inventory.readRead inventory data
inventory.writeCreate/update inventory
inventory.deleteDelete inventory entities
inventory.manageManage advanced inventory states (BLOCK, SOLD, BOOKING…)
⚠️

Common Error Responses

400 Bad Request — Validation failed (ZodError) or invalid input { "error": {...} }
401 Unauthorized — Missing or invalid JWT access token { "error": "Unauthorized" }
403 Forbidden — Authenticated but missing required permission { "error": "Forbidden" }
404 Not Found — Resource does not exist { "error": "Not found" }
500 Internal Server Error — Unexpected server failure { "error": "Internal server error" }
